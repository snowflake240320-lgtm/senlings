# ON_SITE_STATE_MACHINE
## Version: 0.1 | Status: Locked (v0)

## 1. 目的
- 現場内モードの振る舞いを完全固定し、例外処理を明文化する。
- 将来の拡張で壊れない「状態の骨格」を作る。

## 2. 状態定義 (v0)
1. **IDLE**: 起動直後。現場未選択。
2. **MOVE_MODE**: 移動中（現場外）。地図・ナビが主役。
3. **ON_SITE_MODE**: 作業中（現場内）。カメラ・写真が主役。
4. **ARCHIVED**: 現場終了。閲覧専用。

---

## 3. 状態遷移のルール
- **IDLE** → **MOVE_MODE**: 現場選択により自動遷移。
- **MOVE_MODE** → **ON_SITE_MODE**: [到着]ボタン押下（`ARRIVED_ON_SITE` 記録）。
- **ON_SITE_MODE** → **MOVE_MODE**: [終了]ボタン押下（`EXIT_SITE` 記録）。

---

## 4. 例外処理と復元
- **強制終了時**: 再起動時に直前の状態を復元（ON_SITE中に落ちたらON_SITEで復帰）。
- **厳格な排他**: 到着前の撮影禁止。ON_SITE中の他現場選択禁止。

---

## 5. 設計思想
状態は増やさない。ON_SITE中は現場に集中、MOVE中は迷わない。
状態の「けじめ」がデータの純度を作る。
