# MAP_MODE_FLOW
## Version: 0.1 | Status: Locked (v0)

## 1. 目的
- 現場へ迷わず到達する。
- 到着したら即「現場モード（カメラ）」へ入り、無駄な遷移を排除する。

## 2. 二大モード設計（v0固定）
1. **移動モード（現場外）**: 地図が主役。
2. **現場モード（現場内）**: 写真が主役。
※それ以外のモードは作らない。

---

## 3. 各モードの仕様
### 移動モード（現場外）
- **画面**: 現場名、窓口情報、Google Maps、[ナビ開始]、[到着]ボタン。
- **ナビ**: Google Mapsを外部起動。

### 現場モード（現場内）
- **遷移**: [到着]ボタン押下で `ARRIVED_ON_SITE` を記録し、カメラを起動。
- **画面**: 全画面カメラ、[撮影]、[終了]ボタン。
- **保存**: 写真は `{project_id}_{HHMMSS}.jpg` 形式で保存。

---

## 4. 制限とポリシー（v0）
- **手動の徹底**: GPS自動判定や自動カメラ起動は行わない。
- **排他制御**: 到着ボタンを押すまでカメラは使えない。
- **終了処理**: 終了ボタンで「移動モード」へ戻る（勤怠とは分離）。

---

## 5. 設計思想
移動と作業を混ぜない。迷いを消す。
現場に入ったら、撮るだけ。
