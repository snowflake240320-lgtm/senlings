# 彗（SUI）文書群

このディレクトリには、Senlings の最上位思想体系である**彗（SUI）**の文書を保管する。

## 彗とは何か

> 彗（SUI）は、地球規模の結（ゆい）の場を作るための、分散した行為法OSである。

彗は、Senlings の上位ではなく、**循環の核**である。
泥（現場・施工・職人の手）から立ち上ってきた思想が、ここに集まり、また現場に戻る。

Senlings は、彗の**最初の端末「義眼」**として実装される。

| ファイル | 内容 | ステータス |

## 読む順序

1. **spec-sui-history.md** から読む
   - 彗がどこから生まれたか（正史）
   - 何を守り、何を保留するか

2. **spec-sui-charter.md** を読む
   - 開発の最上位ルール
   - 採用・不採用の判断基準

3. **spec-sui-system.md** を読む
   - 八座（やくら）という分散統治
   - Senlings との関係

## Senlings との関係

```
泥（現場・施工・職人の手）
  ↑ 立ち上がる
docs/spec/      ← Senlings: 現場の端末（義眼）
  spec-brand.md
  spec-vision.md（未作成）
  spec-auth.md
  spec-data.md
  spec-4screens.md
  ↑ 立ち上がる
docs/sui/       ← 彗: 思想の循環核
  spec-sui-history.md
  spec-sui-charter.md
  spec-sui-system.md
  ↺ 循環して、更新される
泥に戻る
```

**彗は上位ではなく、循環の核**。
ボトムアップで生まれ、Senlings の現場経験で更新される。

## 唯一のこだわり

> **「今この瞬間に、死なない。」**

これがすべての設計判断の根である。

## 採用・不採用の最終判断

機能の採否は、以下の5項目で判定される（spec-sui-charter.md より）：

1. これは、人を壊さずに判断を続けさせるか
2. これは、関係を壊さず継続できるか
3. これは、誰か一人に利益を集中させていないか
4. これは、失敗しても戻れるか
5. これは、管理ツールになっていないか

## 注記

これらの文書は、整理途中の生きた思想である。
誤字、文の繋がりの不自然さ、未確定の議題が含まれる。

それは欠陥ではなく、**まだ立ち上がり続けている証拠**である。

完璧に整える必要はない。
**作動する法を採用し、答えがまだ閉じないものは議題として保留する**（spec-sui-history.md より）。

## 更新方針

彗の文書は、循環の中で更新される。

更新時に守るべきこと：
- ボトムアップの順序を尊重する（現場で起きたことから立ち上げる）
- 既存の核を壊さない（特に「今この瞬間に、死なない」）
- 八座の拒否権を経る
- 改訂履歴を残す
